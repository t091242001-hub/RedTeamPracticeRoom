在 Linux 系統中，/dev/shm 和 /tmp 雖然都常用於存放臨時數據，但它們在設計目的與硬體層面的運作方式有顯著差異。
什麼是 /dev/shm？
/dev/shm 是 共享記憶體（Shared Memory） 的縮寫。它是一個基於 tmpfs 技術的虛擬檔案系統，直接將資料儲存在物理記憶體（RAM）中，而非硬碟上。 

* 速度極快：因為讀寫操作都在 RAM 進行，速度遠高於傳統硬碟甚至 SSD。
* 揮發性：一旦系統重啟，儲存在此目錄中的資料會全部消失。
* 用途：主要用於進程間通訊（IPC），讓不同的程式能快速共享同一塊記憶體數據，例如 [Oracle 資料庫](https://developer.aliyun.com/article/284318) 就會利用此機制。

------------------------------
與 /tmp 的主要不同

| 特性 | /dev/shm | /tmp |
|---|---|---|
| 儲存媒介 | 物理記憶體 (RAM) | 通常為 磁碟/SSD（部分 Linux 發行版也會掛載為 RAM） |
| 讀寫速度 | 極快，受限於記憶體頻寬 | 較慢，受限於磁碟 I/O 速度 |
| 重啟後資料 | 不保留（一定會消失） | 取決於設定（通常在重啟後會被清除，但存放在磁碟上） |
| 容量限制 | 預設通常為實體記憶體的一半 | 取決於磁碟分割區剩餘空間 |
| 主要目的 | 高性能進程間數據交換 (IPC) | 一般程式運行時的臨時檔案存儲 |

總結建議

* 如果需要極致的讀寫性能，或是在編寫需要多進程共享數據的程式，請使用 /dev/shm。
* 如果只是需要一個地方放暫時性的小檔案（如安裝包解壓縮、編輯器的自動儲存檔），請使用傳統的 /tmp。
