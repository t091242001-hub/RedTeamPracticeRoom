/dev/shm（Shared Memory）是 Linux 下一種基於記憶體的虛擬檔案系統（tmpfs），將檔案直接讀寫至 RAM，速度極快，重啟後資料消失。它主要用於進程間的高速通訊（IPC）。與 /tmp 不同，/dev/shm 專注於高效能記憶體共享，而非通用臨時檔案儲存，且預設大小為記憶體的一半。 


1. /dev/shm 是什麼？
本質： 它是一個 tmpfs (Temporary File System)，掛載在記憶體中的虛擬裝置。
特性： 讀寫速度為閃電般快，因為完全駐留在 RAM 中，不佔用實體硬碟 IO。
用途： 供應用程式進行高效能的共享記憶體運作，例如 IPC（進程間通訊）。
持久性： 重啟系統後，/dev/shm 中的所有檔案將自動消失。
容量： 默認為物理記憶體的一半，但會隨資料量動態調整。 


2. /dev/shm 與 /tmp 的不同
特性 	/dev/shm	/tmp
儲存位置	完全記憶體 (RAM/Swap)	實體硬碟 (HDD/SSD) [通常]
速度	極快 (Memory Speed)	較慢 (Disk Speed)
重啟後	資料遺失 (易失性)	通常遺失 (現代發行版可能為 tmpfs)
主要用途	高效能進程間通信 (IPC)、鎖檔案	通用暫存檔 (應用程式臨時產生的數據)
安全性	需注意記憶體溢出	若填滿會導致系統 IO 問題


3. 總結
應用場景： 如果你需要執行速度極快、且可以隨時丟棄的暫存檔，用 /dev/shm。如果是一般的應用程序臨時文件，用 /tmp。
綁定： 在某些高效能服務場景，為了避開磁碟 I/O 瓶頸，會將 /tmp 直接綁定 (mount --bind) 到 /dev/shm，利用其高速特性。
